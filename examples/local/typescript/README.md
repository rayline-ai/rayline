# Local routing — TypeScript

Route an [`@anthropic-ai/sdk`](https://www.npmjs.com/package/@anthropic-ai/sdk)
call to your **on-device model** through Rayline, mirroring what
`rayline claude --local` does. Traffic flows through Rayline's local proxy on
`127.0.0.1:20810` to the on-device router (`rld`), which routes by your config —
no special headers.

The default local config maps the model name **`rayline-local`** to your
on-device endpoint, so the example just requests that model. Start the router
with `rayline router start` (no Claude session needed); it routes every request
through the router by default, so requesting `rayline-local` reaches the on-device
model.

> **Caveats**
> - The model-name routes (`rayline-local`, `rayline-subagent`) come from
>   Rayline's built-in local config; you can add your own under
>   `routes.model_routes`.
> - A raw API call only gets the model, not Claude Code's agent harness, so the
>   on-device model has no file tools — it answers prompts, it doesn't browse
>   files.

## Setup

```bash
npm install
```

Requires Node 18+ (uses the built-in fetch with an `undici` dispatcher).

## Configure

1. Start the router (give the on-device model a minute to load on first use):

   ```bash
   rayline router start
   ```

   Confirm it's ready:

   ```bash
   curl -s http://127.0.0.1:20810/healthz   # expect "local_available": true
   ```

   Stop it later with `rayline router stop`.

2. Set `CA_PATH` in [`main.ts`](main.ts) to Rayline's proxy CA path for your OS.
   The client must trust this cert because Rayline intercepts TLS:

   | OS | Path |
   | --- | --- |
   | macOS | `~/Library/Application Support/rayline/proxy-ca.pem` |
   | Linux | `~/.config/rayline/proxy-ca.pem` |
   | Windows | `%APPDATA%\rayline\proxy-ca.pem` |

## Run

```bash
npx tsx main.ts
```

You should get a reply generated by your on-device model. For the hosted cloud
router instead (no proxy or CA needed), see
[`../../cloud/typescript`](../../cloud/typescript).
