# Contributing to Rayline Local

Thanks for your interest in Rayline Local, the local-router project in the
broader Rayline ecosystem. This document explains how to engage with the project
during its current phase.

## External Pull Requests Are Not Accepted Yet

Rayline Local is in a locked-down, pre-public-contribution phase. We are not
accepting external pull requests at this time. PRs opened from outside the
maintainer team will be closed without review. This is a temporary policy while
the project's governance, security, and release infrastructure are established.

## Issues Are Welcome

We do welcome issues, including bug reports and feature requests. High-signal,
reproducible reports are the most useful thing you can contribute right now.

When filing an issue, please:

- use the provided issue forms;
- include your OS, Rayline Local version, the exact command you ran, and a
  minimal reproduction;
- describe expected vs. actual behavior;
- never include secrets, API keys, tokens, private prompts, customer data, or
  unsanitized logs.

### AI-Assisted Issues

You may use AI tools, but you must verify the report yourself before filing:
confirm the bug is real and reproducible, and remove any fabricated or
hallucinated details. Please disclose if an issue was substantially
AI-generated.

Unverified AI-generated reports waste reviewer time and may be closed.
Fabricated reports may result in a ban.

## Future Code Contributions

If and when external code contributions are opened, contributors may need to
complete a contributor agreement with copyright assignment to Atlas Futures,
Inc. before substantial pull requests are accepted.

For contribution-policy or contributor-agreement questions, contact
[legal@rayline.ai](mailto:legal@rayline.ai).

## Security

Do not report security vulnerabilities through public issues. See
[SECURITY.md](SECURITY.md) for the private reporting process.
